Concepto base

¿Qué es Streamlit y para qué sirve?

Streamlit es una librería de Python que permite crear aplicaciones web interactivas escribiendo únicamente código Python. No hace falta saber HTML, CSS ni JavaScript.

Si el notebook de Jupyter es tu cuaderno de laboratorio (solo tú lo ves y lo manejas), Streamlit es la ventana al público: convierte ese mismo código en una app que cualquier persona puede usar desde el navegador, sin instalar nada.

Cuándo usar cada herramienta

HerramientaPara quéQuién la usa
Jupyter NotebookExplorar datos, desarrollar y probar códigoSolo el/la desarrollador/a
StreamlitDesplegar una app interactiva lista para usarCualquier usuario final
Flask / DjangoAplicaciones web completas y complejasDesarrolladores web con experiencia

Streamlit es especialmente popular en proyectos de Inteligencia Artificial y Data Science porque permite mostrar modelos, gráficos y resultados de forma inmediata sin hacer frontend.

Paso 1

Instalación y primer arranque

Instalación con pip, igual que cualquier librería de Python:

Terminal
pip install streamlit

Una vez instalado, puedes comprobar que todo funciona lanzando la demo de bienvenida oficial:

Terminal
streamlit hello

Se abrirá automáticamente el navegador con una página de demos interactivas. Si se abre, la instalación fue correcta.

¿Qué versión tengo? Puedes comprobarla con streamlit --version en el terminal. La versión actual es la 1.x y es la que se usa en esta guía.
Paso 2

Crear y ejecutar tu primera app

A diferencia de un notebook, una app de Streamlit es un fichero Python normal (app.py). Lo escribes con cualquier editor (VS Code, PyCharm…) y lo ejecutas desde el terminal.

app.py — la app más sencilla posible
import streamlit as st

st.title("Mi primera app")
st.write("Hola, mundo!")
Terminal — para lanzarla
streamlit run app.py

Streamlit abre automáticamente el navegador en http://localhost:8501. El resultado es:

localhost:8501
Mi primera app
Hola, mundo!

Truco clave: "Always rerun"

Cuando guardas cambios en app.py, Streamlit detecta el cambio y muestra un aviso arriba. Haz clic en "Always rerun" para que la app se recargue automáticamente cada vez que guardes. Así no tienes que recargar manualmente.

Paso 3

Mostrar texto, datos y gráficos

Streamlit tiene funciones específicas para cada tipo de contenido. La más flexible es st.write(): "entiende" lo que le pases y lo muestra de la forma más adecuada.

app.py
import streamlit as st
import pandas as pd
import numpy as np

# Texto
st.title("Mi app de datos")           # Título grande
st.header("Sección 1")                # Encabezado
st.subheader("Subsección")            # Subencabezado
st.write("Texto normal con **negrita**") # Admite Markdown
st.markdown("- Item 1\n- Item 2")     # Markdown completo
st.code("print('hola')", language="python")

# DataFrame
df = pd.DataFrame({
    'Ciudad': ['Madrid', 'Barcelona', 'Sevilla'],
    'Población': [3300000, 1600000, 690000]
})
st.dataframe(df)             # tabla interactiva (con filtros)
st.table(df)                 # tabla estática

# Gráficos rápidos
st.line_chart(np.random.randn(50))
st.bar_chart(df.set_index('Ciudad')['Población'])

# Mapa
map_data = pd.DataFrame(
    np.random.randn(100, 2) / [50, 50] + [40.4, -3.7],
    columns=['lat', 'lon']
)
st.map(map_data)             # mapa centrado en Madrid

Paso 4

Widgets de entrada: sliders, selectores, botones

Los widgets son los controles interactivos de la app. Cuando el usuario mueve un slider o elige una opción, Streamlit re-ejecuta todo el script de arriba a abajo con el nuevo valor. Así funciona la reactividad en Streamlit.

es como una hoja de cálculo. Cuando cambias el valor de una celda, todas las que dependen de ella se actualizan automáticamente. En Streamlit, cambiar un widget relanza todo el script.
app.py
import streamlit as st

# Slider numérico
umbral = st.slider("Valor de umbral", min_value=0, max_value=255, value=127)

# Selector desplegable
operacion = st.selectbox("Elige una operación", ["Canny", "Blur", "Threshold"])

# Casilla de verificación
mostrar_grises = st.checkbox("Mostrar en escala de grises", value=True)

# Botón
if st.button("Procesar imagen"):
    st.write("Procesando...")

# Input de texto
nombre = st.text_input("Nombre del fichero de salida", value="resultado.png")

# Radio buttons
color = st.radio("Color de contorno", ["Verde", "Rojo", "Azul"])

# Los valores se usan igual que variables normales
st.write(f"Umbral seleccionado: {umbral}")
st.write(f"Operación: {operacion}")

Paso 5

Layout: columnas, tabs y sidebar

Por defecto todo en Streamlit se apila verticalmente. Con las herramientas de layout puedes organizar el contenido en columnas, pestañas, acordeones y un panel lateral.

app.py — columnas
# Dos columnas de igual ancho
col1, col2 = st.columns(2)

with col1:
    st.subheader("Imagen original")
    st.image(imagen_original, use_container_width=True)

with col2:
    st.subheader("Imagen procesada")
    st.image(imagen_resultado, use_container_width=True)
app.py — tabs (pestañas)
tab1, tab2, tab3 = st.tabs(["Bordes", "Contornos", "Medición"])

with tab1:
    st.write("Resultado de la detección de bordes")
    st.image(bordes)

with tab2:
    st.write("Contornos detectados")

# Todo lo que se ponga dentro de st.sidebar va al panel izquierdo
with st.sidebar:

Paso 6

Subir ficheros e imágenes

Con st.file_uploader() el usuario puede arrastrar y soltar ficheros directamente en la app. El fichero subido llega como un objeto de Python que puedes leer con las librerías habituales.

app.py — subir una imagen para procesarla con OpenCV
import streamlit as st
import cv2
import numpy as np
from PIL import Image

archivo = st.file_uploader(
    "Sube una imagen",
    type=["jpg", "jpeg", "png"]   # formatos permitidos
)

if archivo is not None:
    # Convertir el fichero subido a un array de NumPy
    img_pil = Image.open(archivo)
    img_np  = np.array(img_pil)           # array RGB
    img_bgr = cv2.cvtColor(img_np, cv2.COLOR_RGB2BGR)

    # Mostrar la imagen original
    st.image(img_np, caption="Imagen cargada", use_container_width=True)

    # Aquí ya puedes procesar img_bgr con OpenCV
    img_gris = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2GRAY)
    st.image(img_gris, caption="Escala de grises", use_container_width=True)
else:
    st.info("Sube una imagen para comenzar.")

Paso 7

Mensajes de estado: éxito, error, aviso

Streamlit tiene mensajes de estado visuales para comunicarle al usuario qué está pasando: si algo fue bien, si hay un error, si hay que esperar…

app.py
# Mensajes de estado
st.success("Imagen procesada correctamente.")
st.error("No se pudo cargar el fichero.")
st.warning("El umbral es muy bajo, puede haber ruido.")
st.info("Sube una imagen para comenzar.")

# Indicador de carga (spinner) durante un proceso largo
with st.spinner("Procesando imagen..."):
    resultado = procesar_imagen(imagen)   # operación lenta

st.success("¡Listo!")

# Barra de progreso
barra = st.progress(0)

Paso 8

Aplicaciones multipágina

Cuando la app crece, conviene dividirla en varias páginas. Streamlit lo hace con una carpeta pages/: cada fichero .py dentro de esa carpeta se convierte automáticamente en una página con su propio enlace en el menú lateral.

mi_proyecto/
├── app.py ← página principal (se ejecuta con streamlit run)
├── pages/
│ ├── 1_Cargar_imagen.py ← página 1
│ ├── 2_Procesar.py ← página 2
│ └── 3_Resultados.py ← página 3
└── requirements.txt
Terminal — se ejecuta igual que siempre
streamlit run app.py

Streamlit detecta la carpeta pages/ automáticamente y crea el menú de navegación lateral. El número al inicio del nombre del fichero controla el orden en el menú.

localhost:8501
Navegación
🏠 Inicio
📁 Cargar imagen
⚙️ Procesar
📊 Resultados
🏠 Página principal
Selecciona una sección en el menú de la izquierda.
Alternativa moderna: desde Streamlit 1.36 también existe st.navigation() para definir páginas directamente en el código, con más control sobre la navegación. Para proyectos sencillos, la carpeta pages/ es más que suficiente.
Paso 9

Caché: no repetir cálculos innecesarios

Recuerda que Streamlit re-ejecuta todo el script cada vez que el usuario interactúa. Si tienes una operación lenta (cargar un modelo de IA, leer un CSV enorme, procesar una imagen pesada), se repetiría en cada clic. La caché evita eso.

es como guardar el resultado de una cuenta complicada en un post-it. La primera vez la calculas, la apuntas, y las siguientes veces miras el post-it en lugar de calcularla de nuevo.
@st.cache_data

Para datos: resultados que se pueden serializar y guardar.

DataFrames listas diccionarios arrays NumPy llamadas a API resultados de cálculos
@st.cache_resource

Para recursos: objetos que no se pueden serializar.

modelos de ML conexiones BD clientes de API objetos grandes únicos
app.py — ejemplo con caché
import streamlit as st
import pandas as pd

# Sin caché: se recargaría el CSV en cada interacción del usuario
# Con caché: se carga una sola vez y se reutiliza

@st.cache_data
def cargar_datos(ruta):
    # Esta función solo se ejecuta la primera vez
    return pd.read_csv(ruta)

@st.cache_resource
def cargar_modelo():
    # Los modelos de IA son lentos de cargar → se cachean con resource
    import pickle
    with open("modelo.pkl", "rb") as f:
        return pickle.load(f)

# Uso: igual que una función normal
df = cargar_datos("datos.csv")
modelo = cargar_modelo()

st.dataframe(df)
⚠️ Nota: el decorador @st.cache (sin sufijo) está obsoleto y eliminado en versiones recientes. Usa siempre @st.cache_data o @st.cache_resource según el caso.
Paso 10

Compartir la app en la nube — gratis

Una vez que la app funciona en local, puedes publicarla en internet de forma gratuita con Streamlit Community Cloud. Cualquier persona podrá usarla sin instalar nada.

Pasos para publicar la app

1

Prepara el repositorio en GitHub

Tu proyecto debe tener estos dos ficheros como mínimo:

mi_proyecto/
├── app.py
└── requirements.txt ← lista de librerías
2

Crea el fichero requirements.txt

requirements.txt
streamlit
opencv-python-headless
numpy
Pillow

Usa opencv-python-headless en la nube (no tiene interfaz gráfica). Localmente puedes usar opencv-python.

3

Sube el proyecto a GitHub

Terminal
git init
git add .
git commit -m "mi app streamlit"
git push origin main
4

Despliega en Streamlit Community Cloud

Ve a share.streamlit.io, inicia sesión con tu cuenta de GitHub, haz clic en "Deploy an app" y pega la URL de tu repositorio. En unos minutos tu app estará online en una URL pública del tipo:

https://tu-usuario-mi-app.streamlit.app
